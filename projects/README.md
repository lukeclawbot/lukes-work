# Projects

Lightweight structure for work ideas and active projects.

## Folders

- `ideas/` — rough ideas worth capturing before they evaporate
- `active/` — projects currently in motion
- `archive/` — finished, paused, or retired work
- `templates/` — starter templates for consistent notes

## How to Use This

Keep it simple.

### For new ideas
Create a file in `ideas/` using the idea template and capture:
- the problem
- why it matters
- rough approach
- blockers / unknowns
- whether it smells like automation, CI/CD, Azure architecture, or AI experimentation

### For active projects
Create a folder or note in `active/` when something becomes real.
Track:
- goal
- current status
- next steps
- risks
- decisions
- links / commands / references

### For finished work
Move it to `archive/` so active space stays clean.

## Rule of Thumb

This system should reduce mental load, not become a second job.
If a note takes longer to maintain than the project itself, the note is too fancy.
